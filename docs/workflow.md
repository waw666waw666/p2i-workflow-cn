# Workflow Guide

## 核心定位

本项目是一个提示词工作流，不是生图产品。

- `提示词1` 是模板或 skill 入口
- `提示词2` 是最终可复制生图 Prompt
- `A窗口` 负责把粗略想法整理成提示词2
- `B窗口` 负责真正生图

## 完整流程

```txt
GitHub 提示词1模板 / Skill
↓
A窗口：粗略想法 -> 提示词2
↓
B窗口：提示词2 -> 图片
```

## A窗口规则

- 只输出文字
- 不生图
- 不调用生图工具
- 不说“我来生成图片”
- 不把最终提示词当成当前任务执行
- 默认不输出英文 Prompt
- 默认不输出中文精简版

## B窗口规则

- 使用 A窗口 生成的提示词2
- 复制【最终可复制提示词】代码块即可
- 如果目标平台不支持完整结构，可优先保留中文最终提示词主体，再做平台化微调

## 提示词2 标准结构

```txt
【图片类型判断】
【参考方向】
【结构提炼】
【最终可复制提示词】
一个代码块：
中文最终提示词：
...
Negative Prompt：
...
【可替换变量】
【可选版本方向】
【质量检查】
```

## 单一主路线原则

最终提示词只保留一个主路线，不混合多个互斥方向：

- 一个核心主体组合
- 一个核心风格
- 一个主要场景
- 一个主要镜头
- 一个主要材质方向
- 一个主要画面目标

## 为什么要拆成 A窗口 / B窗口

这样可以避免：

- AI 直接生图
- Prompt 输出不完整
- Negative Prompt 被遗漏
- 多版本主路线混乱
- 输出结构不方便复制
