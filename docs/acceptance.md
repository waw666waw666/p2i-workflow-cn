# Acceptance Criteria

## 仓库层

- 这是文档型仓库，不包含网页、后端或 API 代码
- 仓库可同时作为开源文档仓库和可复用 skill 发布
- 仓库包含 `README.md`、`SKILL.md`、`LICENSE`、`.gitignore`、`prompts/`、`examples/`、`docs/`

## 术语层

- 只使用两窗口命名：`A窗口`、`B窗口`
- 不再出现旧版 `A/B/C` 三窗口命名
- 不再出现“优化提示词1窗口”这类旧称呼

## 行为层

- `A窗口` 只输出文字，不生图
- `B窗口` 负责真正生图
- `提示词1` 是模板 / skill 入口
- `提示词2` 是最终可复制生图 Prompt

## 输出层

- 最终输出包含 7 个固定部分
- 整个输出只允许 1 个代码块
- 代码块只能出现在【最终可复制提示词】
- 代码块必须同时包含 `中文最终提示词` 和 `Negative Prompt`
- 默认不输出英文 Prompt
- 默认不输出中文精简版

## 内容层

- 不大段复制参考网站原始 Prompt
- 强调提取结构与写法，而不是搬运内容
- 保留单一主路线原则
- 保留可替换变量、可选版本方向、质量检查

## 示例层

- 仓库至少包含 1 个完整示例
- 示例能体现固定结构和唯一代码块规则
